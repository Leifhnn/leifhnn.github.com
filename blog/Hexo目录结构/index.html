学习一个项目时，了解其目录文件的基础作用是至关重要的，这能帮助我们更好地自定义功能。

主要目录介绍

_config.yml

_config.yml 是 Hexo 的最核心配置文件，它决定了博客的全局设置。在这里，你可以配置站点的标题、副标题、作者、语言、时区、URL 等基本信息。例如：

_config.landscape.yml

这个是默认主题的配置文件，主要是调整对应主题的自定义样式和内容。

每个主题都有自己的配置文件，如 themes/your_theme/_config.yml。直接修改它并不明智，因为这会在主题更新时带来麻烦。

最佳实践是：在站点根目录的 _config.yml 中覆盖主题的设置，或者创建一个专门的主题配置文件，如 _config.your_theme.yml，Hexo 会自动合并这些配置。

source/ : 源文件目录

source/ 目录是你存放所有写作内容和资源的地方。主要包括以下几个子目录和文件：

_posts/：文章目录

所有发布的文章（Markdown 文件）都存放在这里。你可以使用 hexo new "My New Post" 命令来创建一篇新文章。每篇文章通常是一个 Markdown 文件，文件名格式为 YYYY-MM-DD-title.md。例如：

独立页面（如 “关于”、”标签页”）

如果你想创建一个 “关于我” 的页面，可以执行 hexo new page "about"。Hexo 会在 source 目录下创建一个 about 文件夹，里面包含一个 index.md 文件，编辑它即可。这个页面的访问路径通常是 your-domain.com/about/。

_data/：数据文件目录

这个目录用于存放数据文件，例如 JSON、YAML 等文件。你可以在主题或插件中引用这些数据文件，以实现动态内容的展示。

themes/：主题目录

themes/ 是存放 Hexo 主题的目录。每个主题都是一个独立的子目录，里面包含了主题的布局（.ejs / .pug）、样式（.css / .styl）、脚本（.js）和它自己的配置文件 _config.yml。

你可以从 GitHub 或 Hexo 官网主题页下载你喜欢的主题，并放到这个目录下。

public/：生成的静态文件目录

当你运行 hexo generate 命令时，Hexo 会将博客的源文件转换为静态文件，并将它们存储在 public/ 目录中。当你执行hexo d，上传GitHub时，就是上传的这个目录。目录里的静态文件可以直接部署到服务器上。

scaffolds/：模板文件目录

scaffolds是“脚手架、骨架”的意思，当你新建文章 (hexo new) 时，Hexo 会根据 scaffolds 目录下的同名模板来创建文件。默认有 post.md、page.md和 draft.md。

你可以编辑这些模板，添加自定义的 Front-Matter 字段（即文章 .md 文件最上方以 --- 分隔的区域），比如预设分类：

node_modules/：项目依赖的模块目录

node_modules/ 是存放 Hexo 项目依赖的 Node.js 模块的目录。一般不用关注。当你安装 Hexo 插件或主题时，相关的模块会被下载到这个目录中。这些模块为 Hexo 提供了各种功能，例如文章渲染、分页、部署等。

总结

在日常使用 Hexo 时，我们主要和 _config.yml (站点配置)、themes/ (主题选择和配置) 以及 source/_posts/ (文章写作) 这几个地方打交道。理解整个目录结构能让你在进行深度定制时更加得心应手。
